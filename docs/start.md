你现在想把 “儒释道技能树 + Infinite Wiki” 走到落地，那就是标准三件套：MVP → 技术栈 → PRD。我帮你按产品经理+开发视角梳理。

⸻

一、MVP（最小可行产品）

目标：用最小成本验证「点亮技能树」+「AI 补链」的核心体验。
	1.	核心体验闭环
	•	用户看到一棵技能树（初版可只放概念节点：仁、义、礼、智、信、道、德、空、无）。
	•	用户点击一个节点 → 节点点亮 → 弹出该概念的释义/经典出处。
	•	AI 推荐延伸节点（例如点“空” → 推荐“色即是空”“无为”）。
	•	用户可继续点亮。
	2.	简化范围
	•	数据范围：
	•	先选三本：儒（论语）、道（道德经）、佛（心经）。
	•	切成概念 + 经典句子，几十个节点即可。
	•	前端界面：
	•	一棵树 or 思维导图，不用太复杂，支持点击/高亮即可。
	3.	成功验证标准
	•	用户能点亮节点。
	•	AI 能给出合理的“延伸推荐”。
	•	用户能回看自己点亮的路径（形成个人技能树）。

⸻

二、技术栈选择

前端（展示技能树）
	•	React + Next.js（主流选择，SSR支持好）
	•	可视化库：
	•	d3.js → 定制化强，但复杂。
	•	react-force-graph → 快速渲染节点网络，交互自然。
	•	MVP 建议：react-force-graph，能快速做出“技能树”效果。

后端（知识图谱 + AI）
	•	数据库：
	•	Neo4j（天然图数据库，适合经文节点/关系）
	•	或简化 → 用 JSON/SQLite 存节点关系（MVP 足够）。
	•	AI 接入：
	•	向量库：Weaviate / Pinecone / FAISS → 存 embedding。
	•	LLM：
	•	生成释义/推荐 → OpenAI GPT-4o / Claude / GLM。
	•	embedding → OpenAI text-embedding-3-large / bge-large-zh。

部署
	•	前端：Vercel（快速上线）。
	•	后端：Supabase / Railway / Node.js Express API。
	•	数据：初版可放 JSON 文件，后期迁到 Neo4j + 向量库。

⸻

三、PRD（产品需求文档 v0.1）

1. 产品愿景

打造一个「AI 驱动的儒释道技能树」：
	•	用户通过点亮节点，建立自己的“经典心智图谱”。
	•	每个节点既是“知识点”，又是“心灵锚点”。
	•	AI 在背后动态补链，让探索无限延伸。

⸻

2. 用户需求
	•	探索感：想把抽象经典变成可见的“技能树”。
	•	成长感：想看到自己逐渐点亮的知识网络。
	•	延伸感：不想止步于死知识，而是无限扩展。
	•	轻负担：不是刷题，而是像玩技能树一样轻松。

⸻

3. 功能模块（MVP）
	•	F1：技能树展示
	•	概念节点以树/图谱形式展示。
	•	未点亮灰色，已点亮高亮。
	•	F2：节点点击 → 释义
	•	用户点击一个节点 → 弹出：
	•	原文
	•	简单释义
	•	出处（经典+章节）
	•	F3：AI 补链推荐
	•	节点被点击后，AI 推荐 2–3 个相关节点。
	•	推荐标准：embedding 相似度 + 知识图谱预置关系。
	•	F4：用户进度记录
	•	记录已点亮节点，形成“个人技能树”。
	•	支持回顾历史路径。

⸻

4. 非功能需求
	•	简洁：MVP 只支持几十个节点，确保不卡顿。
	•	沉浸感：界面极简，不出现多余按钮。
	•	延展性：后续可加更多经典、更多层级。

⸻

5. 演进路线
	•	MVP：概念技能树（30–50 节点）+ 点击释义 + AI 推荐。
	•	V2：支持经典章节 → 节点层级更丰富。
	•	V3：个性化路径 → AI 生成“你的儒释道修行画像”。
	•	V4：支持语音/发呆模式 → AI 自动弹出节点/经句。

⸻

✅ 总结：
	•	MVP → 三本经典 + 概念节点 + 点击释义 + AI 推荐。
	•	技术栈 → React/Next.js + react-force-graph + JSON/Neo4j + embedding + OpenAI。
	•	PRD → 愿景 = 心智技能树；功能 = 点亮、释义、推荐、路径记录。
