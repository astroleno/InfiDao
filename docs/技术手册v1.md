📘 六经注我 — MVP 开发手册（Developer Guide）

版本：v0.1
作者：InfiDao Project
目标：实现“我注六经、六经注我”的语义互注闭环
框架：Next.js + LanceDB + bge-m3 + GLM-4.5 + Vercel

⸻

一、🎯 项目概述

1. 产品定义

“六经注我” 是一个 AI-native 语义互注系统。
用户输入一句话，系统会：
	1.	在《六经》数据库中找到语义相近的段落；
	2.	为匹配结果生成两段互注：
	•	六经注我：经文如何映照我的句子；
	•	我注六经：我的句子如何回应经文；
	3.	生成“延伸链接”，让用户点进下一轮共鸣，构成 Infinite Wiki。

四、🔌 API 规范

1. /api/embed

功能：生成文本嵌入向量
方法：POST
请求

{"text": "我觉得自律很难，但不自律又很痛苦。"}

响应

{"success": true, "embedding": [0.132, -0.081, ...], "model": "bge-m3", "version": "EMB_V1_BGE_M3"}

错误（统一规范）

{"success": false, "error": {"code": "BAD_REQUEST", "message": "..."}}


⸻

2. /api/search

功能：语义检索六经段落

{
  "success": true,
  "results": [

五、🧠 Prompt 模板（核心三段）

1️⃣ retrieve_prompt

用于关键词扩展与检索增强

你是古典文本索引助手。
输入一句现代汉语，输出3~5个关键词或短语，用于匹配儒释道经典原文。
保持文义、象征和比喻层面的一致性。
输入: {{query}}
输出格式: {"keywords":["关键词1","关键词2","关键词3"],"semantic_terms":["概念1","概念2"]}

2️⃣ annotate_prompt

用于生成“双向注解”

你是一位精通《六经》的AI注释者。
请以简洁、诗性但清晰的语言，为给定的现代句子和经文生成两段互注：

【六经注我】——从经文角度回应现代句子  
【我注六经】——从现代句子角度反观经文

要求每段80~120字，语言优雅而通俗。
输入：
现代句子：{{query}}
经文片段：{{passage}}
输出格式：
六经注我：...
我注六经：...

3️⃣ link_reason_prompt

用于生成语义关系类型

请判断两个文本的关系类型，返回 "semantic"（语义相似）、
"contrast"（对比）、或 "symbolic"（象征/隐喻）。
并附上一句简短理由。
输入：
句1：{{query}}
句2：{{passage}}
输出：
{"reason":"semantic","evidence":"两者都谈到自我约束与内心秩序"}

六、💾 部署建议

层级	方案	说明
前端 + API	Vercel	快速部署，Serverless 支持 Route Handlers
数据存储	LanceDB (local)	轻量本地向量库；文件打包即可（≤5–10k 段理想，>10w 段考虑 Supabase+pgvector）
嵌入模型	bge-m3 (via API)	中文检索效果好；可改 text-embedding-3-large
LLM	GLM-4.5（主生产） + Qwen / OpenAI（A/B）	通过 /lib/llm.ts 统一接口
后续迁移	Supabase + pgvector	当数据量 > 10w 段时再切换
国内可用	Cloudflare Pages/Workers 或 Supabase 香港区	GLM-4.5 优先，注意超时与降级

七、📈 MVP 验收标准

目标	验收标准
语义检索	输入现代句后能检索出合理的3~5条经文
注释生成	每条结果生成两段互注，通顺有哲理
延伸探索	点击延伸卡片能再发起新检索
延迟体验	P99 < 3 秒；缓存命中 < 1.2 秒（KV 缓存）
降级策略	模型超时自动降级；检索命中不足时回退关键词检索
可扩展性	LanceDB → Supabase 迁移不改 API
